# Aussie-Mate

> Bilingual (English / 한국어) guide for Korean migrants and visitors settling
> into life in Australia. Bite-size Aussie English phrases, cultural context,
> and practical info for apartments, work, study, and travel.

**Live:** [youraussieguides.com](https://youraussieguides.com)
**Repo:** [github.com/michaelbui92/aussie-mate](https://github.com/michaelbui92/aussie-mate)
**Stack:** Next.js 16 (App Router) · React 19 · Tailwind CSS 4 · TypeScript 5
**Hosting:** Vercel (auto-deploys from `main`)

---

## Quick start

```bash
# Install
npm install

# Dev server (http://localhost:3000)
npm run dev

# Production build
npm run build
npm start

# Lint
npm run lint
```

No env vars required — the site is fully static at build time. All content
is in `app/lib/phrases.ts` (178 Aussie phrases after the dedupe pass that
removed 31 duplicate entries — same phrase was previously listed under
multiple categories with drifting Korean glosses) and the page-level `.tsx`
files.

## Project structure

```
app/
  layout.tsx              Root layout, theme/lang providers, fonts
  page.tsx                Home (hero, featured destinations, intro)
  globals.css             Tailwind v4 @import + @theme + custom CSS
  icon.tsx                Dynamic favicon (Australian flag) via ImageResponse
  robots.ts               /robots.txt (auto-generated by Next.js)
  components/             Reusable UI (Nav, Footer, SearchModal, PhraseExplorer, etc.)
  components/personas/    Persona-targeted content components (BeforeContent, ArrivedContent, HomeContent) — pages under /journey/* import these directly
  context/                ThemeContext (client provider for ThemeProvider)
  lib/                    Data files (phrases, search index, resources, fonts, destinations)
  [page]/page.tsx         One folder per top-level page
  destinations/           Dynamic /destinations/[slug] pages

tests/                    Playwright smoke tests (e2e/)
  smoke.spec.ts           Search modal + home + key pages render
playwright.config.ts      Playwright config
```

### Where to find things

- **Adding a phrase?** Edit `app/lib/phrases.ts` (data), then add a
  Korean search keyword in `app/lib/searchIndex.ts` (search).
- **Adding a page?** Create `app/<slug>/page.tsx` and add the route to the
  appropriate group in `app/components/Nav.tsx` (Home / Living / Explore /
  Learn / More). Add a `metadata` export for SEO.
- **Changing the colour palette?** Edit `@theme { --color-*: ... }` in
  `app/globals.css`. (The old `tailwind.config.ts` was deleted — Tailwind
  v4 uses CSS-first config.)
- **Dark mode behaviour?** Toggled by the theme init script in
  `app/layout.tsx` (adds `.dark` class to `<html>`). The
  `@custom-variant dark` directive in `globals.css` is what makes the
  `dark:` Tailwind utilities respond to that class.

## Bilingual policy

Every page is **English by default**, with a Korean translation toggle
in the top nav. The toggle sets `<html lang="ko">` and switches the
content blocks.

- **English content** is the authoritative source.
- **Korean content** is co-validated with a native speaker (Angela) before
  merging anything user-facing in Korean.
- Search keywords in `app/lib/searchIndex.ts` have both `en` and `ko`
  matches — keep them in sync when adding new entries.

## Deployment

Vercel auto-detects Next.js and deploys on every push to `main`. No
`vercel.json` needed.

To deploy a preview without touching `main`, push to a feature branch
and Vercel creates a preview URL automatically.

## Testing

```bash
# Run Playwright smoke tests (headless)
npx playwright test

# Run with UI
npx playwright test --ui
```

The test suite covers:
- Home page renders
- Search modal opens, accepts input, shows results
- Aussie English page shows the phrase list
- A few key navigation paths work

No unit tests — the site is data-driven and the data is small enough to
spot-check by eye. Playwright smoke tests catch the regressions that
matter (broken pages, broken search).

## Conventions

- **Australian English** everywhere: "colour" not "color", "centre" not
  "center", "organise" not "organize". Exceptions only for code
  identifiers.
- **Sydney/NSW default** for prices, regulations, public-transport
  references. AUD for currency. Flag if a non-Sydney example is given.
- **Voice:** warm, direct, "smart friend" tone. No corporate filler.
